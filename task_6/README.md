# Task 6: Алгоритм Беллмана–Форда

## Описание

Алгоритм поиска **кратчайших путей** из одной вершины во все остальные в направленном взвешенном графе. В отличие от Дейкстры, поддерживает **отрицательные веса** рёбер и **обнаруживает отрицательные циклы**.

## Алгоритм

1. Инициализация: `dist[source] = 0`, остальные `∞`
2. Релаксация: (V−1) проходов по всем рёбрам  
   `dist[v] = min(dist[v], dist[u] + w(u,v))`
3. Дополнительный проход: если любое ребро ещё улучшает путь → есть отрицательный цикл

## Сложность

| Метрика | Значение |
|---------|----------|
| Время | O(V·E) |
| Память | O(V) |

## Ввод

- Количество вершин (нумерация 0..n−1)
- Количество рёбер
- Для каждого ребра: `начало конец вес` (вес может быть отрицательным)
- Стартовая вершина

## Запуск

```bash
python task_6.py
```

Программа выводит пошаговую визуализацию релаксации и итоговые расстояния. При обнаружении отрицательного цикла — соответствующее сообщение.
